# 工作流

## 内容接入

1. 将用户原始资料复制到 `sources/raw/`，不得移动或删除桌面原文件。
2. 使用 `scripts/extract_docx_text.py` 生成可检索的文本副本。
3. 从讲义中保守拆分节点，不擅自改写导师文本。
4. 为节点补充统一元数据、来源状态和版权状态。
5. 在来源不确定时写入 TODO，不填造内容。
6. 老师 Word 稿件存在正文插图时，从 DOCX 原始媒体中保守提取到 `assets/images/teacher_manuscripts/`，按稿件相对位置接入页面，并保留授权检查。

## 英文原文来源优先级

1. 日常英文原文抓取、节点切片和原版插图定位统一使用 `sources/external/principia_english/wikisource_1846_primary/`，主源为 Wikisource 1846 数字页面。
2. 页面引用的译文归属仍为 Andrew Motte 1729 英译本。数字转录页面版本与译本归属必须分开记录。
3. Wikisource 1729 转录页和 Project Gutenberg 1846 仅作为备用与跨源比对。
4. 所有抓取页面必须先缓存在本地，所有节点必须记录来源 URL、抓取时间、提取规则和人工校对状态。
5. 原版页面存在插图时，应下载对应 Wikimedia 图片到 `assets/images/primary_sources/`，同步到 Web 发布目录，并尽量保持原页面中的相对插入位置。
6. 公开发布前必须使用 `sources/external/motte_1729/` 内的 1729 扫描底本核对引用文本，并记录具体扫描页。

## Motte 1729 扫描校核规则

1. `sources/external/motte_1729/` 保留为历史扫描校核库，不再作为日常英文抽取入口。
2. 页码映射、疑难段落核查和必要时的补充 OCR，使用与 IA OCR 逐页匹配的 `sources/external/motte_1729/vol_*/pdf/motte_1729_vol*.pdf`。
3. `sources/external/motte_1729/vol_*/raw_pdf/` 中用户提供的 PDF 是重新排版参考版，仅用于辅助阅读，不作为 1729 页码或英文节点底本。

## 页面建设

1. 先建设“命题1 定理1：扫面积速度恒定”样板页。
2. 以 `docs/standards/sample_page_body_baseline.md` 记录已基本敲定的主体四模块范式。
3. 后续节点优先复用 Hero、英文原文、中译版和现代微积分证明四个主体模块，不回退已确认规则。
4. 动态图示和署名模块按样板页阶段范式复用；思考题、命题网络和评论区暂时保留，继续按待审阅状态迭代。
5. 每个动态演示先在演示文件目录下生成 prompt 文档，写清学术目标、数学模型、视觉结构、交互要求和验收标准，再生成独立 HTML 文件。
6. 网页端必须直接挂载该独立 HTML 文件，保证人工修改 HTML 后页面实时使用同一份演示内容；动态图框区和文字面板应避免互相拖动，桌面端优先使用固定图框与独立滚动说明面板。
7. 动态演示的文字应面向正式课程读者，不使用内部制作说明语气；涉及定律的运动取样必须与数学模型一致；演示 HTML 应包含可核验的数据或对比面板，而不只是动画图形。
8. 署名区使用当前规范：中译版与现代解释、数字化整理与交互演示建设、原典来源、当前页面状态；英文数字主源不在署名区单列。
9. 主体范式确认后再逐章扩展。

## 每次任务收尾

- 更新 `PROJECT_STATUS.md`。
- 更新 `TODO.md` 和 `CHANGELOG.md`。
- 在 `docs/status/` 新建任务记录。
- 对关键技术选择新增 decision log。
- 运行内容状态脚本和网站构建检查。
