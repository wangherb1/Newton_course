# 动态演示 HTML 生成指令：命题1 定理1 等面积定律

## 文件接口

- 演示目录固定为：`web/public/demos/book1_chapter02/prop001_theorem001/`
- 生成指令固定保留为：`equal_area_demo.prompt.md`
- 网页端挂载文件固定保留为：`equal_area_demo.html`
- 外部或人工优化稿可继续保存在同目录，例如：`equal_area_optimized.html`

此接口必须保留。后续如果自动生成质量不足，可以继续把本 prompt 交给其他模型优化，再将结果回填到同目录并融合到 `equal_area_demo.html`。

## 目标

为《原理》第一卷第二章命题1定理1生成一个可直接嵌入课程网页的独立 HTML 动态演示。演示对象不是一般椭圆运动动画，而是“固定力心、焦点连线、相等时间、相等扫过面积”这一核心结论。

## 必须学习的结构优点

本模块应学习 `equal_area_optimized.html` 的主体结构，而不是只学习其图形外观：

- 顶部或面板首段给出命题标题与核心陈述。
- 交互控制区必须保留“等时间分段数、播放、暂停、下一段、显示全轨道、重置”等完整控制。
- 几何与面积检验区必须显示离心率、每段理论面积、曲线采样最大误差、当前运行时间。
- 力心连线特征对比区必须用表格并列比较近焦点段与远焦点段的平均连线长度、扫过转角、多边形积分面积。
- 图例必须说明固定力心 S、几何中心 C、当前相等时间扫过区域。
- 读图说明必须解释“等时间”来自平近点角等间隔，而不是等转角。

## 学术叙述要求

- 页面文字面向正式课程读者，采用客观学术陈述语气。
- 不使用“给用户看”“给你看”“当前原型”“交互演示说明”等内部制作语气。
- 标题、图例、数据面板和读图要点共同服务于命题事实：物体受指向固定点的向心力作用时，力心连线扫过面积与时间成正比。
- 可以使用椭圆轨道作为形象演示，但必须说明固定力心位于椭圆焦点处。

## 数学模型

- 椭圆半长轴为 `a`，半短轴为 `b`，离心率为 `e`。
- 固定力心 `S` 位于椭圆的一个焦点。
- 轨道点不得按等角度或等真近点角取样。
- 相等时间取样必须使用平近点角：

```text
M = E - e sin(E)
```

- 以 `M` 等间隔增加。由于 `M` 与时间成正比，每个时间段 `Delta t` 对应相同的 `Delta M`。
- 焦点扇形面积满足：

```text
Delta A = ab Delta M / 2
```

- 每段面积必须相等；近焦点段表现为连线较短、转角较大，远焦点段表现为连线较长、转角较小。

## 视觉与布局要求

- 左侧图框区使用课程页风格：深色网格背景、金色面积扇形、浅色椭圆轨道。
- 图框区必须固定，不随右侧文字区滚动；右侧面板应独立滚动。
- 桌面端采用左图右文布局；移动端采用上图下文布局，图框仍固定在演示 HTML 的上部，面板独立滚动。
- 不得出现横向裁切；移动端按钮、滑条、表格和标签必须完整可读。
- 默认状态应显示全轨道等面积扇区，使读者无需先操作即可看到定律整体结构。

## 交互要求

- 提供播放、暂停、重置、下一段和显示全轨道控制。
- 允许调整等时间分段数，最大值至少为 `36`。
- 动画播放时逐段显示等面积扇形。
- “下一段”从全轨道状态进入时应显示第一段，不得跳过第一段。

## 校验信息

- 面板中应给出每段理论面积。
- 面板中应给出多边形采样面积的最大误差。
- 面板中应并列比较近焦点段和远焦点段的平均半径、转角和面积。
- 面积、半径、转角的数值显示必须与图形一致。

## 文件组织

- HTML 文件为单文件交付，包含必要 CSS 和 JavaScript。
- 不依赖外部 CDN。
- 文件可直接通过 iframe 嵌入课程网页。
- 同目录保留本 prompt 文档，便于后续人工修改或重新生成演示。

## 验收标准

- 桌面和窄屏均能清楚看到椭圆、焦点、当前物体、焦点连线和扫过面积。
- 不得出现等角速度误导。
- 不得出现面积明显不等而文字声称相等的情况。
- 右侧面板滚动时左侧图框区保持固定。
- 页面语言是正式课程内容的一部分，而不是内部制作说明。
- 构建通过后，课程页面与独立 HTML 文件展示同一个演示内容。
